# Foodzy

An AI-powered kitchen assistant. Add the ingredients you have, pick a cuisine
(including Indian regional styles like North Indian, Maharashtrian, Lucknowi,
Kerala, and more), and get recipe ideas generated by Google Gemini. Sign in
with Gmail and save your favorite dishes to revisit later.

Built with **Next.js (App Router) + Tailwind CSS**.

## Features

- AI dish ideas + full recipes via the Gemini API (`/api/generate-ideas`, `/api/generate-recipe`)
- Indian cuisine with regional sub-styles (North Indian, South Indian, Maharashtrian, Lucknowi/Awadhi, Kerala, Bengali, Punjabi, Gujarati, Hyderabadi, Goan)
- Gmail sign-in (Google Identity Services, with a simple Gmail-address fallback if no OAuth client ID is configured)
- Save dishes and revisit them later on the **Saved dishes** page

## Getting started locally

```bash
npm install
cp .env.example .env.local   # then fill in GEMINI_API_KEY
npm run dev
```

Open http://localhost:3000. You'll land on `/login` first — sign in, then
you'll be redirected to the dashboard.

## Environment variables

| Variable | Required | Notes |
|---|---|---|
| `GEMINI_API_KEY` | Yes | Server-side only. Get one from Google AI Studio (https://aistudio.google.com/apikey). Used by the API routes to call Gemini. |
| `NEXT_PUBLIC_GOOGLE_CLIENT_ID` | No | Enables real Google one-tap sign-in. Get it from Google Cloud Console → APIs & Services → Credentials → OAuth client ID (Web application). Without it, `/login` falls back to a simple Gmail-address form. |

## Deploying

### 1. Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

### 2. Deploy on Vercel

1. Go to vercel.com/new and import your GitHub repo.
2. Vercel auto-detects Next.js — no build settings to change.
3. Under **Environment Variables**, add:
   - `GEMINI_API_KEY` = your Gemini API key
   - `NEXT_PUBLIC_GOOGLE_CLIENT_ID` = your Google OAuth client ID (optional)
4. If you set up `NEXT_PUBLIC_GOOGLE_CLIENT_ID`, add your Vercel domain
   (e.g. `https://your-app.vercel.app`) to **Authorized JavaScript origins**
   in the Google Cloud Console OAuth client settings.
5. Click **Deploy**.

That's it — every push to `main` will auto-redeploy.

## Notes

- Saved dishes and the signed-in user are stored in the browser's
  `localStorage` — there's no backend database yet, so saved dishes are
  per-device/browser.
